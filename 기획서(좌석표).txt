정원호 [기획서] 

좌석표 : 	로그인 후 좌석을 선택할 할 수 있는 gui를 제공한다.
		좌석 선택 후 바로 사용 or 예약을 선택할 수 있게 하고 
		결제창으로 넘어가도록 진행한다. 그러나 시간이 남아 있다면 바로 이용하도록 진행한다.

순서 및 구현 기능:

공간 선택: 자신이 공부하기 원하는 공간을 선택 (라이브러리존, 카페존, 스터디룸)/ 공간 선택 후 자신이 원하는 공간의 좌석 Map을 불러온다.
											

라이브러리존(독서실형태),카페존(카페형 공간) : 	1. 자신이 원하는 위치의 좌석 번호를 선택합니다.(번호별 버튼으로 구현할 예정)
									2. 버튼 클릭 하면 바로 사용 or 예약 버튼으로 이동
									3. 남은 시간이 충분하다면 바로 이용, 시간이 부족하다면 결제창으로 이동.  
									4. 사용이 시작되면 좌석표의 색상이 변하고, 남은 시간을 나타냅니다.

스터디룸(회의실 or 스터디 모임): 1. 자신이 원하는 위치의 스터디 룸을 선택합니다.(4인 이상의 룸이기 때문에 가격을 어떻게 정할지 정하면 좋겠습니다.)
						2. 스터디룸은 100프로 예약제로 서비스를 제공합니다. 정기권 사용이 불가능합니다. 바로 결제창으로 이동합니다.(제가 임의로 작성한 부분인데, 같이 고민해보면 좋겠습니다.)
						3. 사용이 시작되면 좌석표의 색상이 변하고, 남은 시간을 나타냅니다.
						
좌석변경 : 1. 좌석변경란 클릭. (좌석표 상단 메뉴 선택 창에 배치!)
		2. 바로 사용가능 좌석이 표시되고 선택하면 적용.!


사물함 선택: 	1. 사물함 선택란 팝업!
			2. 사물함 선택 후 결제창으로 이동.

구현 기능:
	1. 이미지(도면)을 판넬 뒤에 삽입, ImageIcon클래스 사용
	2. 클릭시 예약 또는 즉시 사용으로 화면 전환, setVisible(True) 
	3. 좌석 배치 후 색상 변경 button.setBackground(Color.색상)
	4. 남은 시간 버튼에 텍스트로 알림 JButton(text, new ImageIcon("image"));
		텍스트 위치 button .setVerticalTextPosition(JButton.BOTTOM);//아래로
	5. 도면마다 필요한 버튼을 불러올 class 작성 필요!

구현시 사용할 변수 이름:
	좌석표 : st(seating_table)
	공간 선택 버튼:
		라이브러리존 : lb_btn 카페존 : cf_btn, 스터디룸 : sr_btn
	
	라이브러리존 좌석 번호 : lb_btn1~34
	카페존 좌석 번호 : cf_btn1~16
	스터디룸 좌석 : sr_btn 1~2
	
	좌석변경 버튼: seat_change

예외처리:
	한 사용자가 중복으로 예약하지 못하도록 예약 좌석을 array로 만들어 추가 하지 못하게 하겠습니다.
